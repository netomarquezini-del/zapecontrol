# Story 1.2 — Gestão de Usuários

**Epic:** 1 — MVP
**Status:** Draft
**Agente:** @dev

## Objetivo
Permitir que o Admin crie, edite e desative usuários com perfis de acesso.

## Acceptance Criteria

- [ ] Página `/settings/users` com listagem de usuários
- [ ] Botão "Novo Usuário" abre dialog com formulário
- [ ] Campos: nome, email, senha, perfil (admin/gerente/operador)
- [ ] Validação de email único
- [ ] Edição de usuário existente (nome, perfil, reset senha)
- [ ] Desativação de usuário (soft delete)
- [ ] Badge visual para perfil (cores diferentes por role)
- [ ] Acesso restrito: somente Admin vê menu "Usuários"
- [ ] Permissão verificada server-side na API route

## Notas Técnicas
- API routes em `src/app/api/users/`
- Usar shadcn Dialog para formulário
- Permissão checada via `permissions.ts`

## Dependências
- Story 1.1 (auth + layout base)
