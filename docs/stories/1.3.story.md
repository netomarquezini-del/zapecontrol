# Story 1.3 — Cadastro de Produtos

**Epic:** 1 — MVP
**Status:** Draft
**Agente:** @dev

## Objetivo
Permitir cadastro, listagem, edição e inativação de produtos com upload de foto.

## Acceptance Criteria

- [ ] Tabela `products` criada com migration
- [ ] Bucket `product-photos` configurado no Supabase Storage
- [ ] Página `/products` com listagem em grid/tabela
- [ ] Busca por SKU ou nome (filtro client-side)
- [ ] Botão "Novo Produto" abre formulário
- [ ] Campos: SKU (único), nome, foto (upload), tipo (embalado/desembalado)
- [ ] Upload de foto com preview antes de salvar (max 5MB, JPG/PNG/WebP)
- [ ] Edição de produto existente
- [ ] Inativação de produto (soft delete com toggle)
- [ ] Card do produto mostra: foto, nome, SKU, badge tipo
- [ ] Acesso: Admin e Gerente podem cadastrar/editar. Operador vê listagem readonly.

## Notas Técnicas
- Upload via Supabase Storage client
- Comprimir imagem client-side antes de upload (< 1MB)
- Usar `next/image` para exibição otimizada
- RLS: todos lêem, admin/gerente escrevem

## Dependências
- Story 1.1 (auth + layout)
