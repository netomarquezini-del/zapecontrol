# Story 1.4 — Lançamento de Produção

**Epic:** 1 — MVP
**Status:** Draft
**Agente:** @dev

## Objetivo
Permitir lançamento diário de produção vinculado a produtos cadastrados, com histórico.

## Acceptance Criteria

- [ ] Tabela `production_entries` criada com migration
- [ ] Página `/production` com duas abas: "Lançar" e "Histórico"
- [ ] Aba "Lançar":
  - [ ] Campo data (default: hoje)
  - [ ] Busca de produto por SKU ou nome (autocomplete com Command)
  - [ ] Ao selecionar produto: exibir foto, nome, SKU, badge tipo
  - [ ] Campo quantidade (inteiro positivo)
  - [ ] Botão "Adicionar" — permite múltiplas linhas antes de salvar
  - [ ] Lista de itens adicionados com total
  - [ ] Botão "Salvar Lançamento" — grava todos os itens
- [ ] Aba "Histórico":
  - [ ] Listagem de lançamentos com filtro por data e produto
  - [ ] Colunas: data, produto (foto+nome+SKU), quantidade, lançado por
  - [ ] Edição de lançamento (mesmo dia, pelo autor ou Admin)
  - [ ] Exclusão de lançamento (somente Admin)
- [ ] Registro automático de `created_by` e timestamp
- [ ] Acesso: Todos os perfis podem lançar

## Notas Técnicas
- Autocomplete com shadcn Command component
- Batch insert na API para múltiplos itens
- Filtro de histórico server-side com paginação

## Dependências
- Story 1.1 (auth)
- Story 1.3 (produtos cadastrados)
