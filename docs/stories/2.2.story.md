# Story 2.2 — Metas e Regras de Pontuação

**Epic:** 2 — Core
**Status:** Draft
**Agente:** @dev

## Objetivo
Sistema de metas configuráveis com regras de pontuação flexíveis por cenário.

## Acceptance Criteria

- [ ] Tabelas `scoring_rules` e `goals` criadas com migration
- [ ] Página `/goals` com duas abas: "Metas" e "Regras de Pontuação"
- [ ] Aba "Regras de Pontuação":
  - [ ] Listagem de regras ativas
  - [ ] CRUD de regras com campos: nome, tipo condição, valor condição, pontos, prioridade
  - [ ] Tipos de condição: `product_type`, `specific_sku`
  - [ ] Regras padrão pré-criadas: embalado=1.0, desembalado=0.5
  - [ ] Indicador de prioridade (maior = mais específica)
  - [ ] Preview: "Se produto X → Y pontos por unidade"
- [ ] Aba "Metas":
  - [ ] Definição de meta global (diária, semanal, mensal em pontos)
  - [ ] Definição de meta por produto (opcional)
  - [ ] Vigência (data início, data fim opcional)
  - [ ] Histórico de metas anteriores
- [ ] Indicador visual de atingimento: 🟢 >=100%, 🟡 70-99%, 🔴 <70%
- [ ] Acesso: Admin configura regras. Admin/Gerente definem metas.

## Notas Técnicas
- Scoring engine em `src/lib/scoring.ts`
- Regras resolvidas por prioridade DESC, primeira match ganha
- Fallback: 1.0 ponto se nenhuma regra match
- Seed com regras padrão
