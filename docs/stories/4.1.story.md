# Story 4.1 — Perfil Fábrica e CRUD de Pedidos

**Epic:** 4 — Pedidos vs Produção
**Status:** Draft
**Agente:** @dev
**Dependências:** 1.1 (Auth), 1.3 (Produtos)

## Objetivo
Adicionar o perfil "fábrica" ao sistema de roles e implementar a criação/gestão de pedidos consolidados enviados do CD para a fábrica.

## Acceptance Criteria

- [ ] Role `fabrica` adicionado ao check constraint da tabela users
- [ ] Permissões do role `fabrica` configuradas em `permissions.ts`
- [ ] Menu lateral adaptado para o perfil fábrica (só vê Pedidos e Dashboard Pedidos)
- [ ] Migration `007_create_orders.sql` aplicada (tabelas orders + order_items)
- [ ] Tipos TypeScript: `Order`, `OrderItem`, `OrderStatus`, `Priority` definidos
- [ ] API `POST /api/orders` — criar pedido com múltiplos itens
- [ ] API `GET /api/orders` — listar pedidos com filtros (status, período)
- [ ] API `GET /api/orders/[id]` — detalhe do pedido com itens e progresso
- [ ] API `PUT /api/orders/[id]` — editar pedido (somente status aberto)
- [ ] API `PATCH /api/orders/[id]/cancel` — cancelar pedido (somente Admin)
- [ ] Código do pedido gerado automaticamente (PED-{ANO}-{SEQ})
- [ ] Página `/orders` — listagem de pedidos com filtros e status visual
- [ ] Página `/orders/new` — criar pedido com seleção de produtos e quantidades
- [ ] Página `/orders/[id]` — detalhe do pedido com progresso por item
- [ ] Autocomplete de produto (busca por SKU ou nome) com foto/preview
- [ ] Adicionar/remover múltiplas linhas de itens no pedido
- [ ] Validação: quantidade > 0, produto obrigatório, ao menos 1 item

## Notas Técnicas
- Reutilizar componente de busca de produtos (autocomplete) similar ao de produção
- Status do pedido é calculado automaticamente via trigger no banco
- Sidebar deve mostrar ícone de "Pedidos" para admin, gerente e fábrica
- Perfil fábrica NÃO pode criar pedidos, apenas visualizar
