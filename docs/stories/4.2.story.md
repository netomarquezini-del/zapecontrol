# Story 4.2 — Lançamento de Entregas Parciais

**Epic:** 4 — Pedidos vs Produção
**Status:** Draft
**Agente:** @dev
**Dependências:** 4.1 (Pedidos)

## Objetivo
Implementar o sistema de lançamento de entregas diárias pela fábrica, com controle de saldo e atualização automática do status do pedido.

## Acceptance Criteria

- [ ] Migration `008_create_deliveries.sql` aplicada (tabela deliveries + view + trigger)
- [ ] Tipos TypeScript: `Delivery` definido
- [ ] API `POST /api/orders/[id]/deliveries` — lançar entrega com múltiplos itens
- [ ] API `GET /api/orders/[id]/deliveries` — histórico de entregas do pedido
- [ ] API `PUT /api/orders/[id]/deliveries/[deliveryId]` — editar entrega (mesmo dia, autor ou Admin)
- [ ] Validação: quantidade entregue não pode exceder saldo pendente do item
- [ ] Página `/orders/[id]/deliver` — tela de lançamento de entrega
- [ ] Na tela de entrega, exibir para cada item: produto (foto, nome, SKU), qty pedida, já entregue, saldo, prioridade
- [ ] Input de quantidade entregue por item (só itens com saldo > 0 aparecem)
- [ ] Badge visual de prioridade: normal (sem badge), urgente (⚡), crítico (🔴)
- [ ] Após salvar entrega, status do pedido atualiza automaticamente (aberto → parcial → concluído)
- [ ] Toast de confirmação ao salvar entrega
- [ ] Perfil `fabrica` pode lançar entregas
- [ ] Perfil `fabrica` NÃO pode editar entregas de outros ou de dias anteriores
- [ ] Histórico de entregas visível na página de detalhe do pedido (timeline)

## Notas Técnicas
- View `order_items_progress` no banco calcula saldo automaticamente
- Trigger `update_order_status` atualiza status do pedido após cada entrega
- Reutilizar padrão de formulário multi-linha do lançamento de produção
- Data da entrega default: hoje (pode alterar para mesmo dia)
