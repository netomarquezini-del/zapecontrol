# Story 4.3 — Dashboard de Acompanhamento de Pedidos

**Epic:** 4 — Pedidos vs Produção
**Status:** Draft
**Agente:** @dev
**Dependências:** 4.1 (Pedidos), 4.2 (Entregas)

## Objetivo
Criar o dashboard visual de acompanhamento de pedidos com visão por pedido, visão consolidada por produto e indicadores de progresso em tempo real.

## Acceptance Criteria

- [ ] API `GET /api/orders/dashboard` — retorna stats consolidadas:
  - Total pedidos abertos
  - Total itens pendentes (saldo > 0)
  - Progresso geral (% de todos pedidos abertos)
  - Total itens urgentes/críticos pendentes
- [ ] API `GET /api/orders/dashboard/consolidated` — visão consolidada por produto:
  - Produto, total pedido (soma de todos pedidos), total entregue, falta, nº pedidos
- [ ] Página `/orders/dashboard` — dashboard de acompanhamento
- [ ] Cards resumo no topo: Pedidos Abertos, Itens Pendentes, Progresso Geral (%), Urgentes
- [ ] Visão por Pedido (tabela):
  - Código, data, status, barra de progresso (%), botão expandir
  - Ao expandir: itens com produto, pedido, entregue, falta, barra, prioridade
- [ ] Visão Consolidada (tabela agrupada por produto):
  - Produto (foto + nome), total pedido, total entregue, falta, nº pedidos
  - Ordenado por saldo pendente (maior primeiro)
  - Destaque visual para itens urgentes/críticos
- [ ] Toggle entre Visão por Pedido e Visão Consolidada (tabs)
- [ ] Filtros: status (aberto/parcial/concluído/todos), período, produto, prioridade
- [ ] Barra de progresso colorida por item (gradiente verde conforme avança)
- [ ] Cards com cores: verde (>80%), amarelo (50-80%), vermelho (<50%)
- [ ] Responsivo (desktop prioritário)
- [ ] Acessível por: admin, gerente, fábrica

## Notas Técnicas
- Utilizar a view `order_items_progress` do banco para queries eficientes
- Reutilizar componentes de card/stats do dashboard principal (M5)
- Usar Recharts para barras de progresso se necessário
- Tabs com shadcn/ui Tabs component
