# Story 4.4 — Sistema de Prioridades e Alertas

**Epic:** 4 — Pedidos vs Produção
**Status:** Draft
**Agente:** @dev
**Dependências:** 4.1 (Pedidos), 4.3 (Dashboard)

## Objetivo
Implementar o sistema de prioridades por item do pedido com gestão visual, alteração de prioridade e destaque no dashboard da fábrica.

## Acceptance Criteria

- [ ] API `PATCH /api/orders/[id]/items/[itemId]/priority` — alterar prioridade do item
- [ ] Somente Admin e Gerente podem alterar prioridades
- [ ] Na página de detalhe do pedido, botão para alterar prioridade de cada item
- [ ] Dropdown de prioridade: Normal, Urgente (⚡), Crítico (🔴)
- [ ] No dashboard de pedidos, itens urgentes/críticos aparecem destacados:
  - Borda colorida (laranja para urgente, vermelho para crítico)
  - Badge de prioridade visível
  - Seção separada "Itens Prioritários" no topo do dashboard
- [ ] Na tela de entrega da fábrica, itens ordenados por prioridade (crítico → urgente → normal)
- [ ] Card "Urgentes" no dashboard mostra contagem de itens urgentes + críticos
- [ ] Sidebar mostra badge de contagem se há itens críticos pendentes
- [ ] Ao mudar prioridade, registrar timestamp da alteração

## Notas Técnicas
- Prioridade é por item (order_item), não por pedido inteiro
- Ordenação: critico (peso 3) > urgente (peso 2) > normal (peso 1)
- Badge no sidebar: reutilizar padrão de notification badge
