## Objetivo
Criar toda a estrutura de dados (migrations Supabase), tipos TypeScript, permissões e navegação base para o módulo SDR Station.

## User Story
> Como SDR, quero que a estrutura de dados do módulo esteja pronta, para que todas as funcionalidades subsequentes tenham onde persistir informações.

- [ ] Migration criada para tabela `sdr_messages` (lead_id, sdr_id, direction, content, channel [whatsapp|instagram|phone], media_url, status, external_id, created_at)

## Integration Verification
- IV1: Tabelas existentes do zapecontrol permanecem inalteradas
- IV2: Sidebar existente funciona normalmente com a nova seção
- IV3: Sistema de auth/permissões existente continua operando

## Notas Técnicas
- Usar prefixo `sdr_` em todas as tabelas para isolamento
- Campo `channel` em `sdr_messages` usa enum: whatsapp, instagram, phone
- Campo `status` em `sdr_leads` usa enum: novo, tentativa, conectado, qualificado, agendado, descartado
- Pipeline status no lead, não em tabela separada
- JSONB para `steps` em cadências permite flexibilidade na definição de sequências
