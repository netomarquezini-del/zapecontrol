# Story 5.10 — Copilot Real-Time

**Epic:** 5 — SDR Station
**Status:** Draft
**Executor:** @dev
**Quality Gate:** @architect
**Quality Gate Tools:** [code_review, websocket_validation, performance_test]
**Dependências:** 5.9
**PRD:** docs/prd-sdr-station.md

## Objetivo
Implementar copilot AI que fornece sugestões em tempo real durante as ligações do SDR.

## User Story
> Como SDR, quero receber sugestões do AI em tempo real durante a ligação, para melhorar minha performance e tratar objeções com mais confiança.

## Acceptance Criteria

- [ ] API `GET /api/sdr/copilot/stream` — endpoint WebSocket/SSE para streaming de sugestões
- [ ] API `POST /api/sdr/copilot/context` — enviar contexto inicial da chamada (dados do lead)
- [ ] Componente `<CopilotPanel />` visível durante a chamada (lateral ao softphone)
- [ ] Streaming de sugestões baseado no áudio da conversa via Claude API (streaming response)
- [ ] Detecção de objeções com contra-argumentos sugeridos em tempo real
- [ ] Sugestão de próximos passos baseado no contexto da conversa
- [ ] Dados do lead visíveis durante a chamada: nome, empresa, cargo, histórico de interações, notas
- [ ] Copilot modelado a partir do copilot do closer existente (adaptar prompts)
- [ ] Toggle para ativar/desativar copilot por preferência do SDR
- [ ] Copilot não interfere na qualidade de áudio da chamada
- [ ] Latência de sugestão < 3 segundos entre fala detectada e sugestão exibida
- [ ] Painel mostra: sugestões ativas, objeções detectadas, dados do lead, script sugerido

## Integration Verification
- IV1: WebSocket/SSE estável durante chamadas de até 30 minutos
- IV2: Latência de sugestão < 3 segundos
- IV3: Copilot não impacta performance do softphone WebRTC

## Notas Técnicas
- Abordagem 1 (recomendada): Twilio Media Streams → captura áudio em real-time → Whisper streaming → Claude streaming
- Abordagem 2 (simplificada): Capturar áudio local do browser → enviar chunks → processar
- Claude API com streaming (`stream: true`) para exibir sugestões progressivamente
- Prompt do copilot: incluir contexto do lead, script base do SDR, objeções comuns e contra-argumentos
- Adaptar prompt do copilot do closer (já existente no sistema)
- Toggle salvo como preferência do usuário (localStorage ou tabela de settings)
- Custo: Claude Haiku para real-time (mais rápido e barato) ou Sonnet para qualidade superior
- Esta é a story mais complexa tecnicamente — considerar MVP com transcrição por chunks (a cada 10s) em vez de streaming contínuo

## Testing Guidance
- **Approach:** Integration + Manual
- WebSocket/SSE: conexão estabelecida e mantida durante chamada simulada
- Contexto: dados do lead carregados corretamente no painel copilot
- Sugestões: enviar trecho de áudio → verificar sugestão retornada em < 3s
- Objeções: enviar áudio com objeção conhecida → contra-argumento sugerido
- Toggle: desativar copilot → nenhum processamento ocorre
- Estabilidade: manter conexão por 30 minutos sem queda
- Performance: copilot não degrada qualidade de áudio do softphone
