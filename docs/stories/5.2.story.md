# Story 5.2 — CRM Core: Leads e Pipeline

**Epic:** 5 — SDR Station
**Status:** Draft
**Executor:** @dev
**Quality Gate:** @architect
**Quality Gate Tools:** [code_review, pattern_validation, performance_test]
**Dependências:** 5.1
**PRD:** docs/prd-sdr-station.md

## Objetivo
Implementar o gerenciamento completo de leads com pipeline kanban, import CSV, filtros e timeline.

## User Story
> Como SDR, quero gerenciar meus leads em um pipeline visual, para ter controle total sobre o status de cada contato sem depender do CLINT.

## Acceptance Criteria

- [ ] API `POST /api/sdr/leads` — criar lead
- [ ] API `GET /api/sdr/leads` — listar leads com filtros (status, tags, sdr, data, busca)
- [ ] API `GET /api/sdr/leads/[id]` — detalhe do lead com timeline
- [ ] API `PUT /api/sdr/leads/[id]` — editar lead
- [ ] API `DELETE /api/sdr/leads/[id]` — deletar lead
- [ ] API `PATCH /api/sdr/leads/[id]/status` — mover lead no pipeline
- [ ] API `POST /api/sdr/leads/import` — import CSV com mapeamento de colunas
- [ ] Página `/comercial/sdr-station/leads` — listagem com filtros, busca por nome/telefone/email
- [ ] Página `/comercial/sdr-station/pipeline` — kanban com drag-and-drop (Novo → Tentativa → Conectado → Qualificado → Agendado → Descartado)
- [ ] Componente de detalhe do lead com timeline unificada (vazia nesta story, preenchida nas seguintes)
- [ ] Sistema de tags customizáveis (criar, atribuir, remover, filtrar)
- [ ] Import CSV: upload, preview, mapeamento de colunas, validação, import em batch
- [ ] Contadores por status no pipeline (quantos leads em cada etapa)

## Integration Verification
- IV1: Dados de leads persistem corretamente no Supabase com RLS
- IV2: Permissões respeitam o usuário logado
- IV3: Performance aceitável com 5.000+ leads (paginação)

## Notas Técnicas
- Kanban pode usar @dnd-kit ou react-beautiful-dnd
- Import CSV em batch (100 por vez) para não travar UI
- Timeline é componente reutilizável que será alimentado por calls, messages e notes nas stories seguintes
- Busca usa ilike no Supabase para nome, telefone e email

## Testing Guidance
- **Approach:** Integration + E2E
- CRUD leads: criar, ler, editar, deletar — verificar persistência
- Import CSV: upload com colunas corretas e incorretas, validar tratamento de erro
- Pipeline: drag-and-drop atualiza status no banco
- Filtros: combinação de status + tags + busca retorna resultados corretos
- Paginação: performance com 5.000+ leads (query < 500ms)
- RLS: usuário só vê leads atribuídos a ele (ou todos se admin)
