# Story 5.3 — Integração Twilio: Setup e Pool de Números

**Epic:** 5 — SDR Station
**Status:** Draft
**Executor:** @dev
**Quality Gate:** @architect
**Quality Gate Tools:** [security_scan, api_validation, credential_review]
**Dependências:** 5.1
**PRD:** docs/prd-sdr-station.md

## Objetivo
Configurar a integração com Twilio e implementar o gerenciamento do pool de números com BINA por DDD.

## User Story
> Como gestor, quero configurar os números Twilio e o pool de BINA por DDD, para que o dialer use o número local correto para cada lead.

## Acceptance Criteria

- [ ] API `GET /api/sdr/numbers` — listar números do pool
- [ ] API `POST /api/sdr/numbers` — cadastrar número Twilio com DDD
- [ ] API `PUT /api/sdr/numbers/[id]` — editar número (status, DDD)
- [ ] API `DELETE /api/sdr/numbers/[id]` — remover número do pool
- [ ] API `POST /api/sdr/numbers/test` — teste de chamada (liga para número informado)
- [ ] API `POST /api/sdr/numbers/validate` — validar credenciais Twilio
- [ ] Lógica de seleção BINA: dado DDD do lead, selecionar número local com rodízio
- [ ] Fallback: se não houver número do DDD do lead, usar número padrão
- [ ] Página `/comercial/sdr-station/numeros` — CRUD de números com status visual
- [ ] Indicadores por número: total de chamadas, última utilização, status (ativo/pausado/bloqueado)
- [ ] Configuração de credenciais Twilio via variáveis de ambiente (TWILIO_ACCOUNT_SID, TWILIO_AUTH_TOKEN, TWILIO_API_KEY, TWILIO_API_SECRET)
- [ ] Validação de que credenciais estão configuradas antes de permitir operações

## Integration Verification
- IV1: Credenciais Twilio armazenadas em env vars, nunca no banco
- IV2: Seleção de BINA funciona para DDDs mapeados e fallback
- IV3: Nenhum impacto nas rotas API existentes do zapecontrol

## Notas Técnicas
- Usar Twilio Node.js SDK (`twilio` package)
- Rodízio de números: round-robin por DDD, incrementar call_count
- Status bloqueado = número reportado como spam, precisa ser substituído
- Teste de chamada usa Twilio REST API para fazer uma chamada curta e verificar conectividade

## Testing Guidance
- **Approach:** Integration + Manual
- Validação de credenciais: testar com credenciais válidas e inválidas
- CRUD números: criar, editar status, deletar
- Seleção BINA: dado lead com DDD 11, sistema seleciona número (11)
- Seleção BINA fallback: dado lead com DDD sem número mapeado, usa número padrão
- Rodízio: após N chamadas no número 1, próxima usa número 2 do mesmo DDD
- Teste de chamada: verificar que Twilio API recebe request correto (mock em dev)
