# Story 5.4 — Power Dialer: Discagem Simultânea

**Epic:** 5 — SDR Station
**Status:** Draft
**Executor:** @dev
**Quality Gate:** @architect
**Quality Gate Tools:** [code_review, webrtc_validation, webhook_security]
**Dependências:** 5.2, 5.3
**PRD:** docs/prd-sdr-station.md

## Objetivo
Implementar o power dialer com 5 linhas simultâneas, softphone WebRTC, conexão automática e disposição rápida.

## User Story
> Como SDR, quero disparar 5 chamadas simultaneamente e ser conectado automaticamente quando alguém atender, para maximizar meu tempo produtivo.

## Acceptance Criteria

- [ ] API `POST /api/sdr/calls/dial` — disparar até 5 chamadas simultâneas para próximos leads da fila
- [ ] API `POST /api/sdr/calls/webhook` — receber status de chamada do Twilio (ringing, answered, completed, no-answer, busy, failed)
- [ ] API `POST /api/sdr/calls/hangup` — derrubar chamadas pendentes quando uma é atendida
- [ ] API `POST /api/sdr/calls/[id]/disposition` — salvar disposição pós-chamada
- [ ] Componente `<Softphone />` flutuante (bottom-right) com WebRTC via Twilio Client SDK
- [ ] Botão "Discar Próximos" seleciona próximos 5 leads da fila (por cadência ou manual)
- [ ] BINA aplicada automaticamente via lógica da Story 5.3
- [ ] Ao lead atender: sistema cria conference room, conecta lead + SDR, derruba outras 4 chamadas
- [ ] Latência entre atendimento e conexão com SDR < 500ms
- [ ] Status visual de cada chamada em tempo real (discando → tocando → atendida/não atendida)
- [ ] Modal de disposição rápida após encerrar chamada: Atendeu / Não atendeu / Agendar / Sem interesse / Número errado / Caixa postal
- [ ] Disposição salva na tabela `sdr_calls` e cria entrada na `sdr_interactions` (timeline)
- [ ] Notificação visual quando lead liga de volta (chamada inbound) — badge + som
- [ ] Log completo de cada chamada (lead_id, sdr_id, duration, twilio_sid, disposition, number_used)
- [ ] Webhook Twilio com validação de signature (X-Twilio-Signature)

## Integration Verification
- IV1: WebRTC funciona em Chrome e Edge (desktop)
- IV2: Webhooks Twilio processados com validação de signature
- IV3: Timeline do lead atualizada em tempo real após chamada
- IV4: Softphone persiste durante navegação entre páginas do SDR Station

## Notas Técnicas
- Twilio Client SDK para WebRTC: `@twilio/voice-sdk`
- Conference approach: criar TwiML conference room, adicionar SDR como participant, conectar lead atendido
- Para derrubar chamadas: usar `client.calls(sid).update({ status: 'completed' })` nas não-atendidas
- Softphone como componente no layout do SDR Station, não como página separada
- Inbound calls: configurar número Twilio para webhook que notifica via WebSocket/SSE
- Fila de discagem: leads ordenados por cadence_execution.next_action_at ou manual

## Testing Guidance
- **Approach:** Integration + E2E + Manual
- Disparo de chamadas: verificar que 5 chamadas são criadas via Twilio API (mock em dev)
- Webhook atendimento: simular callback Twilio com status "answered" → SDR conectado
- Webhook não-atendimento: simular "no-answer" → chamada derrubada
- Signature validation: rejeitar webhooks com signature inválida
- Disposição: salvar disposição atualiza `sdr_calls` e cria `sdr_interactions`
- Softphone: WebRTC conecta e áudio funciona (teste manual em Chrome)
- BINA: verificar que número correto é usado por lead
