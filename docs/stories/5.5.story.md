# Story 5.5 — Gravação e Compliance

**Epic:** 5 — SDR Station
**Status:** Draft
**Executor:** @dev
**Quality Gate:** @architect
**Quality Gate Tools:** [security_scan, compliance_review, storage_validation]
**Dependências:** 5.4
**PRD:** docs/prd-sdr-station.md

## Objetivo
Implementar gravação automática de chamadas, controle de horários ANATEL e compliance LGPD.

## User Story
> Como gestor, quero que todas as chamadas sejam gravadas automaticamente e dentro da legislação, para ter histórico e base para análise de performance.

## Acceptance Criteria

- [ ] Gravação automática habilitada em todas as chamadas via Twilio Recording API (`record: true` no TwiML)
- [ ] API `POST /api/sdr/calls/recording` — webhook de recording completed (recebe URL do áudio)
- [ ] Áudio baixado do Twilio e armazenado no Supabase Storage (bucket `sdr-recordings`)
- [ ] Link do áudio salvo na tabela `sdr_calls` (campo `recording_url`)
- [ ] Controle de horários ANATEL implementado:
  - Seg-Sex: 8h-21h
  - Sáb: 8h-13h
  - Dom e feriados: bloqueado
  - Timezone-aware (America/Sao_Paulo)
- [ ] Botão "Discar" desabilitado fora dos horários permitidos com mensagem explicativa
- [ ] Caller ID válido em todas as chamadas (número do pool, nunca anônimo)
- [ ] Aviso de gravação no início da chamada (TwiML `<Say>` ou áudio pré-gravado antes de conectar)
- [ ] Página `/comercial/sdr-station/gravacoes` — listagem com player de áudio inline
- [ ] Filtros na página de gravações: por data, SDR, lead, duração mínima
- [ ] Política de retenção: gravações com mais de 90 dias marcadas para review (configurável)
- [ ] URLs de áudio assinadas (signed URLs) com expiração para segurança

## Integration Verification
- IV1: Áudios acessíveis via Supabase Storage com URLs assinadas
- IV2: Bloqueio de horário impede discagem fora dos horários ANATEL
- IV3: Armazenamento de áudio não impacta performance do Supabase
- IV4: Aviso de gravação audível antes da conversa iniciar

## Notas Técnicas
- Twilio recording callback: quando gravação termina, Twilio envia URL do áudio via webhook
- Download do áudio do Twilio e upload para Supabase Storage para não depender de retenção Twilio
- Feriados nacionais: usar lista fixa dos feriados federais + configuração para feriados locais
- Player de áudio: HTML5 `<audio>` com controles nativos (play, pause, seek, speed)
- Signed URLs com expiração de 1 hora para acesso seguro

## Testing Guidance
- **Approach:** Integration + Manual
- Gravação: verificar que TwiML inclui `record: true`
- Webhook recording: simular callback com URL do áudio → download → upload Supabase Storage
- Horários ANATEL: tentar discar fora do horário → bloqueado com mensagem
- Horários ANATEL: discar dentro do horário → permitido
- Feriados: verificar bloqueio em data de feriado nacional
- Aviso de gravação: verificar que TwiML `<Say>` executa antes da conversa
- Signed URLs: URL expira após 1 hora
- Player de áudio: play, pause, seek funcional
