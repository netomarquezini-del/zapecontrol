# Story 5.6 — Inbox Multicanal (WhatsApp + Instagram)

**Epic:** 5 — SDR Station
**Status:** Draft
**Executor:** @dev
**Quality Gate:** @architect
**Quality Gate Tools:** [code_review, webhook_security, api_validation]
**Dependências:** 5.2
**PRD:** docs/prd-sdr-station.md

## Objetivo
Implementar inbox bidirecional para WhatsApp e Instagram Direct integrado ao workspace do SDR.

## User Story
> Como SDR, quero enviar e receber mensagens por WhatsApp e Instagram Direct no mesmo painel, para atender o lead no canal que ele preferir.

## Acceptance Criteria

### WhatsApp
- [ ] API `POST /api/sdr/whatsapp/send` — enviar mensagem (texto, áudio)
- [ ] API `POST /api/sdr/whatsapp/send-template` — enviar template HSM com variáveis
- [ ] API `POST /api/sdr/whatsapp/webhook` — receber mensagens inbound + status updates
- [ ] Integração com Evolution API (ou Twilio WhatsApp) para envio/recebimento
- [ ] Número da empresa como remetente

### Instagram
- [ ] API `POST /api/sdr/instagram/send` — enviar mensagem (texto, imagem)
- [ ] API `POST /api/sdr/instagram/webhook` — receber DMs inbound via Meta webhook
- [ ] Integração com Meta Graph API para Instagram Direct
- [ ] Conta business como remetente
- [ ] Vinculação de conta Instagram ao lead (match por username ou manual)

### Inbox Unificado
- [ ] Componente `<MultichannelInbox />` no workspace do SDR
- [ ] Lista de conversas com indicador de canal (ícone WhatsApp / Instagram)
- [ ] Chat view com bolhas (enviado/recebido), timestamps, status de entrega
- [ ] Envio de texto em ambos os canais
- [ ] Envio de áudio via WhatsApp
- [ ] Envio de imagens via Instagram
- [ ] Templates de mensagem pré-configurados com variáveis ({{nome}}, {{empresa}}, etc.)
- [ ] CRUD de templates: criar, editar, deletar, listar
- [ ] Indicador de mensagens não lidas (badge no menu + na conversa)
- [ ] Status de entrega: enviado ✓, entregue ✓✓, lido (azul) — quando disponível pelo provider
- [ ] Filtro por canal no inbox (Todos / WhatsApp / Instagram)
- [ ] Mensagens salvas na tabela `sdr_messages` com channel correto
- [ ] Cada mensagem cria entrada na `sdr_interactions` (timeline do lead)

## Integration Verification
- IV1: Webhooks WhatsApp e Instagram processados sem conflito com outras rotas
- IV2: Mensagens persistem com channel correto e aparecem na timeline do lead
- IV3: Templates renderizam variáveis corretamente em ambos os canais

## Notas Técnicas
- WhatsApp via Evolution API: já pode ter infra existente no zapecontrol — verificar
- Instagram Direct API requer Meta Business Suite + App Review (processo iniciado em 5.1)
- Se aprovação Instagram atrasar, implementar WhatsApp primeiro, Instagram como feature flag
- Rate limits Instagram: mais restritivos, implementar fila de envio com throttling
- Real-time: usar polling curto (5s) ou WebSocket para atualizar inbox sem refresh
- Templates WhatsApp HSM precisam ser aprovados pela Meta antes de usar
- Templates Instagram não existem nativamente — usar templates internos apenas

## Testing Guidance
- **Approach:** Integration + E2E + Manual
- WhatsApp envio: verificar que Evolution API/Twilio recebe mensagem corretamente (mock em dev)
- WhatsApp recebimento: simular webhook inbound → mensagem salva em `sdr_messages` com channel=whatsapp
- Instagram envio: verificar request para Meta Graph API (mock em dev)
- Instagram recebimento: simular webhook Meta → mensagem salva com channel=instagram
- Templates: renderização de variáveis ({{nome}} → "João Silva")
- Timeline: mensagem aparece na timeline do lead após envio/recebimento
- Status de entrega: verificar atualização de status via webhook
- Inbox: filtro por canal funciona (Todos / WhatsApp / Instagram)
- Badge não-lido: incrementa ao receber, zera ao abrir conversa
