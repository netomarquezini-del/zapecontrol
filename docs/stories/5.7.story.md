# Story 5.7 — Cadência Automática

**Epic:** 5 — SDR Station
**Status:** Draft
**Executor:** @dev
**Quality Gate:** @architect
**Quality Gate Tools:** [code_review, scheduler_validation, integration_test]
**Dependências:** 5.4, 5.6
**PRD:** docs/prd-sdr-station.md

## Objetivo
Implementar o motor de cadência automática que executa a sequência de contato (ligações + WhatsApp/Instagram) sem intervenção manual do SDR.

## User Story
> Como SDR, quero que o sistema execute automaticamente a sequência de contato (3 ligações → WhatsApp/Instagram), para não precisar lembrar e decidir manualmente o próximo passo.

## Acceptance Criteria

- [ ] API `GET /api/sdr/cadences` — listar cadências configuradas
- [ ] API `POST /api/sdr/cadences` — criar cadência com steps
- [ ] API `PUT /api/sdr/cadences/[id]` — editar cadência
- [ ] API `DELETE /api/sdr/cadences/[id]` — deletar cadência (soft delete)
- [ ] API `POST /api/sdr/cadences/execute` — endpoint do motor de execução (chamado por cron)
- [ ] API `PATCH /api/sdr/cadences/executions/[id]/pause` — pausar execução de um lead
- [ ] API `PATCH /api/sdr/cadences/executions/[id]/resume` — retomar execução
- [ ] API `PATCH /api/sdr/cadences/executions/[id]/skip` — pular etapa atual
- [ ] Cadência padrão criada: Dia 1 liga, Dia 2 liga, Dia 4 liga, Dia 5 WhatsApp, Dia 6 Instagram
- [ ] Página `/comercial/sdr-station/cadencias` — builder visual de cadência
- [ ] Builder: adicionar/remover/reordenar steps, definir canal (telefone/whatsapp/instagram), intervalo entre steps
- [ ] Motor de execução (Supabase Edge Function ou cron via API):
  - Processa `sdr_cadence_executions` onde `next_action_at <= now()` e `status = active`
  - Para step tipo "call": adiciona lead na fila de discagem
  - Para step tipo "whatsapp/instagram": dispara mensagem do template configurado
  - Respeita horários ANATEL (não executa fora do permitido, reagenda para próximo horário válido)
- [ ] Lead entra automaticamente na cadência ao ser criado ou importado
- [ ] Lead sai da cadência quando: atende ligação, agenda reunião, pede para não ligar, completa todas as etapas, ou é descartado manualmente
- [ ] Dashboard de cadência: quantos leads em cada etapa, quantos pausados, quantos completaram
- [ ] Suporte a múltiplos canais por step (telefone, WhatsApp, Instagram)

## Integration Verification
- IV1: Scheduler executa dentro dos horários corretos (timezone America/Sao_Paulo)
- IV2: Cadência integra com dialer (5.4) para steps de ligação e inbox (5.6) para mensagens
- IV3: Performance aceitável com 1.000+ leads em cadência simultânea

## Notas Técnicas
- Motor de execução: Supabase Edge Function com cron (pg_cron) ou Vercel Cron Jobs
- Intervalo de execução do cron: a cada 5 minutos
- Steps da cadência armazenados como JSONB: `[{ step: 1, channel: "phone", delay_days: 0, template_id: null }, ...]`
- Quando lead atende ou agenda, trigger automático via disposition (Story 5.4) remove da cadência
- Fila de discagem: leads do motor de cadência são priorizados no "Discar Próximos"

## Testing Guidance
- **Approach:** Integration + Scheduler
- Cadência CRUD: criar, editar, deletar cadência
- Builder: adicionar/remover steps, reordenar, salvar JSONB correto
- Motor execução: simular cron → leads com next_action_at <= now() processados
- Step telefone: lead adicionado à fila de discagem
- Step WhatsApp/Instagram: mensagem disparada via template
- Horários: scheduler não executa fora de horário ANATEL, reagenda
- Saída automática: lead que atende sai da cadência
- Saída automática: lead que agenda sai da cadência
- Pause/resume: pausar lead mantém posição, retomar continua do step correto
- Performance: processar 1.000 leads em cadência < 30 segundos
