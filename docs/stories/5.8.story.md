# Story 5.8 — Agenda e Handoff para Closer

**Epic:** 5 — SDR Station
**Status:** Draft
**Executor:** @dev
**Quality Gate:** @architect
**Quality Gate Tools:** [code_review, integration_test, workflow_validation]
**Dependências:** 5.2, 5.7
**PRD:** docs/prd-sdr-station.md

## Objetivo
Implementar o sistema de agendamento de reuniões com closers e follow-up automático até a data da call.

## User Story
> Como SDR, quero agendar reuniões na agenda do closer e acompanhar o follow-up automaticamente, para garantir que o lead compareça.

## Acceptance Criteria

- [ ] API `POST /api/sdr/schedules` — criar agendamento (lead_id, closer_id, datetime, notes)
- [ ] API `GET /api/sdr/schedules` — listar agendamentos com filtros (closer, data, status)
- [ ] API `PUT /api/sdr/schedules/[id]` — editar agendamento
- [ ] API `PATCH /api/sdr/schedules/[id]/status` — atualizar status (agendado → confirmado → realizado / no-show)
- [ ] Ao marcar disposição "Agendar" no modal pós-call (Story 5.4), abrir formulário de agendamento
- [ ] Formulário: seleção de closer (dropdown), data/hora, observações
- [ ] Agendamento salvo na tabela `sdr_schedules`
- [ ] Lead move automaticamente para status "Agendado" no pipeline
- [ ] Follow-up automático via WhatsApp: D-1 (lembrete véspera) e D-0 (confirmação no dia)
- [ ] Follow-up usa templates configuráveis com variáveis ({{nome}}, {{data}}, {{hora}}, {{closer}})
- [ ] Visualização de agenda por closer (calendário ou lista)
- [ ] Indicador de próximos agendamentos no workspace do SDR (hoje e amanhã)
- [ ] Registro de no-show: se reunião não é marcada como "realizado" após 1h do horário, marcar como no-show
- [ ] Entrada na timeline do lead ao criar agendamento

## Integration Verification
- IV1: Agendamentos persistem corretamente no Supabase
- IV2: Follow-up automático usa o motor de cadência (Story 5.7) ou scheduler independente
- IV3: Status do pipeline atualiza automaticamente ao agendar

## Notas Técnicas
- Follow-up D-1 e D-0: criar cadence_execution específica para follow-up ou usar cron dedicado
- Closers: usar tabela users existente com role = closer (ou admin)
- Calendário: componente simples (lista por dia), não precisa de integração Google Calendar inicialmente
- No-show detection: cron que verifica schedules com datetime < now() - 1h e status != realizado

## Testing Guidance
- **Approach:** Integration + E2E
- Agendamento: criar via disposição "Agendar" → salva em `sdr_schedules`
- Pipeline: lead move para "Agendado" automaticamente
- Follow-up D-1: verificar que WhatsApp é disparado na véspera
- Follow-up D-0: verificar que WhatsApp é disparado no dia
- No-show: schedule com datetime passado e status != realizado → marcado como no-show
- Agenda closer: listar agendamentos por closer e data
- Timeline: agendamento aparece na timeline do lead
