# Story 5.9 — Transcrição e AI Analysis

**Epic:** 5 — SDR Station
**Status:** Draft
**Executor:** @dev
**Quality Gate:** @architect
**Quality Gate Tools:** [code_review, api_validation, cost_monitoring]
**Dependências:** 5.5
**PRD:** docs/prd-sdr-station.md

## Objetivo
Implementar transcrição automática de chamadas via Whisper e análise AI via Claude API.

## User Story
> Como gestor, quero que as chamadas sejam transcritas e analisadas por AI automaticamente, para avaliar performance sem ouvir cada gravação.

## Acceptance Criteria

- [ ] API `POST /api/sdr/transcriptions/process` — processar transcrição de uma chamada (chamado após gravação finalizar)
- [ ] API `GET /api/sdr/transcriptions/[call_id]` — obter transcrição e análise de uma chamada
- [ ] Após chamada finalizada e gravação salva (webhook Story 5.5), disparar processamento em background
- [ ] Transcrição via OpenAI Whisper API (modelo large-v3) para PT-BR
- [ ] Transcrição salva na tabela `sdr_transcriptions` (campo `text`)
- [ ] Análise via Claude API gera:
  - Resumo da ligação (3-5 frases)
  - Score de qualidade (1-10) com justificativa
  - Objeções identificadas e como foram tratadas (JSONB)
  - Pontos de melhoria para o SDR (JSONB)
  - Próximos passos identificados
- [ ] Análise salva na tabela `sdr_transcriptions` (campos `summary`, `score`, `objections`, `improvements`)
- [ ] Processamento em background — SDR não é bloqueado
- [ ] Status de processamento visível: pendente → transcrevendo → analisando → concluído / erro
- [ ] Tempo máximo de processamento: 5 minutos (timeout com retry)
- [ ] Página de gravações (Story 5.5) enriquecida: transcrição + análise lado a lado com player
- [ ] Custo estimado por transcrição monitorável (log de tokens/minutos consumidos)

## Integration Verification
- IV1: Transcrição PT-BR com qualidade aceitável
- IV2: Custo por transcrição dentro do orçamento (~R$0,01/min Whisper + ~R$0,02 Claude)
- IV3: Análise AI consistente e útil para coaching

## Notas Técnicas
- Whisper API: enviar áudio como arquivo, recebe texto transcrito
- Áudio do Supabase Storage → download → enviar para Whisper → salvar transcrição
- Claude API prompt para análise: incluir contexto do lead (empresa, cargo) + transcrição completa
- Background processing: Supabase Edge Function async ou Vercel Serverless com timeout maior
- Retry: se Whisper falhar, tentar novamente após 30s (máx 3 tentativas)
- Custo: Whisper ~$0.006/min, Claude Haiku para análise ~$0.01/call — muito acessível

## Testing Guidance
- **Approach:** Integration + Manual
- Transcrição: enviar áudio de teste PT-BR → verificar texto retornado coerente
- Análise AI: enviar transcrição de teste → verificar resumo, score, objeções preenchidos
- Background: processamento não bloqueia SDR (async)
- Status: verificar progressão pendente → transcrevendo → analisando → concluído
- Timeout: simular falha Whisper → retry após 30s (máx 3)
- Custo: logar tokens/minutos consumidos por transcrição
- UI: transcrição + análise aparecem lado a lado com player na página de gravações
